# 英伟达财报前的一些思考

> 原文作者：[Gavin Baker (@GavinSBaker)](https://x.com/GavinSBaker) · 2026年2月25日
>
> 原文链接：[X/Twitter](https://x.com/gavinsbaker/status/2026719209158271230)

以下是一些可能对关注英伟达的人有所帮助的思考。在我看来，对这家公司最重要的动态是 **需求的持续性** 和 **AI 的投资回报率**，后者与 GPU 的使用寿命密切相关。

## 1. 需求持续性：这次会不会不一样？

从需求持续性的角度看，市场历史上每一个可比案例都表明，我们最终会迎来一场金融泡沫和过度建设。Carlota Perez 在她的经典著作《技术革命与金融资本》（*Technological Revolutions and Financial Capital*）中对此有精彩阐述。她的核心论点是：每当出现技术革命——铁路、无线电或互联网——金融市场会正确理解其潜力，随之而来的兴奋导致泡沫（如 Mauboussin 所解释的，这是由观点多样性崩溃驱动的）。泡沫导致过度建设，过度建设导致需求暂时下降，进而引发崩盘，然后使能技术的过剩供给催生出"黄金时代"。这正是互联网时代发生的事情。

对英伟达而言，真正重要的是潜在的泡沫和过度建设，而非本季度的盈利或下季度的指引——买方对这些数字早已充分了解。**盈利持续性才是英伟达的一切，而不是今年的增长。** 英伟达的估值可以理解为：市场本质上在表达其相信，由泡沫推动的过度建设将使英伟达盈利接近局部峰值。显然，英伟达的股价并不存在估值泡沫——市场担忧的是基本面泡沫，即资本支出泡沫而非估值泡沫。如果市场对英伟达 2027 财年之后能保持高个位数收入复合增长率建立信心，这可能对其估值产生正面影响。

## 2. 瓦特和晶圆的短缺可能阻止过度建设

相信"这次不一样"是危险的，但我认为这次可能真的不一样。**全世界根本性地缺乏瓦特（电力）和晶圆**，解决这些短缺可能需要数年。瓦特和晶圆的短缺可能阻止过度建设——超大规模企业如果可以，一定会过度建设，但他们根本做不到。据我所知，Perez 书中提到的先例技术部署过程中，并不存在可比的短缺来阻止部署。没有过度建设就很难出现崩盘，尤其是在当前科技股与必需消费品股估值倍数相当的水平上。

**晶圆可能比电力更重要**，作为使 AI 周期更平滑、更持久的瓶颈。台积电由一群务实、强硬、老派的半导体老将掌管，他们视自己的任期为对国家瑰宝的管理，是为台湾人民守护张忠谋的遗产。如果没有瓦特和晶圆的约束，英伟达未来 24 个月可能会增长得快得多，但随后过度建设和崩盘就变得更加可能。黄仁勋可能希望台积电更快地扩产，但我认为台积电的克制对所有参与者都是好事。即使我们奇迹般地解决了电力短缺，这些务实的老将也可能单枪匹马地阻止过度建设。**AI 的晶圆密集性可能反而是它的救星。**

## 3. 更平滑的 AI 周期对人类是好事

我的最佳估计是，要实现 Citrini 描述的那种不太可能的假设场景，我们大约需要 **1000 倍以上的算力**，而达到那里所需的时间将给人类留出调整和最大化 AI 诸多潜在收益的空间。詹姆斯·瓦特发明旋转蒸汽机后，花了超过 50 年火车才广泛取代马匹。AI 应该会更快，但我们仍有时间适应和调整。而且人类有一个巨大优势：只需 20-30 瓦就能实现"通用智能"，这在电力受限的世界中将持续是一个优势。因此，更平滑、更持久的 AI 周期不仅对金融市场有利，对人类也是好事。

## 4. GPU 租赁价格反映了 Token 的经济价值

GPU 的租赁价格是 **"AI 投资回报率"的心跳**。我们应该预期随着更强大、更高效的 GPU 不断推出，租赁价格会随时间下降——即使 AI 的投资回报率非常正面。然而，近四年前推出的 H100 的租赁价格在过去两个月内大幅飙升，这表明 **Agentic AI，特别是 Agentic Coding，正在创造显著的经济价值**。而且许多新模型——ChatGPT 5.2、Codex 5.3 和 Grok 4.20——只是 Blackwell 训练运行的早期检查点，远未完成和优化。

即使是 6 年前的 A100，据 AWS 反馈，利用率仍然满载，每小时定价保持坚挺。我本以为 Blackwell 推出后情况会相反。这强烈暗示 **GPU 的使用寿命至少为 6 年**，这比大多数客户的折旧周期还要长。这意味着 GPU 的融资利率应该进一步下降，因为残值可能高于预期。

相比 ASIC，这一点很重要——为单一模型或应用开发优化的 ASIC 在当前变革速度下不太可能有如此长的使用寿命。这意味着 ASIC 更难融资，资本成本更高。**专业化可能是使用寿命的敌人，而 GPU 更灵活、可编程的本质可能是它的朋友。** 使用 GPU 进行预填充（prefill）和即将推出的基于 SRAM 的芯片进行解码（decode），应该会进一步延长 GPU 相对于 ASIC 的使用寿命。最终，这不再仅仅是 GPU 的事——你需要 8-9 个芯片协同工作的交响乐，而不仅仅是单个 ASIC。

题外话：我打赌黄仁勋肯定后悔把 DPU 这个名字用了——"Decode Processing Unit"多好——但他一定会想出一个很棒的名字。

## 5. 预填充与解码的分离可能给英伟达生态带来成本优势

预填充与解码的分离（disaggregation），在英伟达生态中可能比 TPU 生态更早实现。再加上 Google 对 TPU v8 的保守设计选择和英伟达对 Rubin 的激进选择，英伟达客户可能首次获得相对于 Google 的显著成本优势。Google 目前缺乏交换式规模化网络架构（switched scale-up networking fabric）来支持 TPU，这在当前主导前沿的 MoE 架构推理方面，可能进一步加大英伟达生态的成本优势。

Google 很可能一直在利用其 TPUv7 的成本优势来压低 Token 价格，试图从 Anthropic、OpenAI 和 xAI 等竞争性独立实验室那里吸走经济氧气。这种成本优势的潜在终结应该会促使 Google 做出更理性的决策——即使是他们也无法长期承受几百亿的 Token 低价亏损——这对 AI 投资回报率应该是非常正面的，尤其是当 Blackwell 集群从训练转向推理的时候。这可能是黄仁勋团队难以公开讨论的话题，但确实是一个重要的动态。

## 6. 一个小小的愿望

最后，我对英伟达有一个小愿望：希望他们重新使用超级英雄别名作为代号。令人惊讶的是，"绿色战队"居然从来没用过 **"Banner"** 作为代号。 🟢
