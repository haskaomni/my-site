# Karri Saarinen：关于 AI 的一些笔记

> 原文：Karri Saarinen / @karrisaarinen, [Some Notes on AI](https://x.com/karrisaarinen/status/2048267794924650791)  
> 说明：以下为中文翻译，尽量保留原文语气与段落结构，少量调整中文表达以便阅读。

我们正接近上一轮模型编程能力大幅跃升之后的第六个月。六个月通常是蜜月期。之后，一些现实就会开始浮现。

我对 AI 的态度通常是谨慎乐观。多数时候，我试着活在当下的现实里，而不是活在某个遥远未来：AGI 脱离人类，形成自己的 TechnoCore 社会。

能力是真实的，限制也是真实的。更有意思的问题，不是“这一切会走向哪里？”，而是“现在到底有什么真的不同了？”

我觉得当前围绕 AI 的市场叙事过于简单，说实话也有点令人沮丧。讨论里常常容不下乐观主义和末日论之间的中间地带。

我更关心的正是这个中间地带：到底什么正在改变，什么是有用的，什么被过度炒作，什么带来真实风险，以及我们仍然不理解什么。

## 计划一文不值，但规划至关重要

最近，计划似乎正在变得不时髦。

我通常思考 AI 影响的方式，是问同一个决策或活动在 AI 之前是否也能完成。据我所知，从来没有一个硬性要求说组织必须做很长周期的计划。如果说有什么共识，反而一直是要更敏捷。可即便如此，许多公司最后还是形成了年度或半年度计划周期。

所以问题是：为什么？这些计划周期到底在解决什么问题？AI 真的解决了那个问题吗？

我没有完整答案，但我常常觉得，计划的重点不在输出物，也就是那些计划本身，而在于对齐和承诺这项练习。它迫使组织聚到一起，讨论什么真正重要，决定优先级，创造意义，然后把这个方向传达给所有人。

有时候，计划也是为了提前处理组织复杂性或边界问题；计划创造出一块共享地形，让不同部门可以在上面导航。计划本身其实只是一个强制函数，让这些会议和讨论得以发生。

所以，虽然 AI 可能改变时间线并提高带宽，但我还没有完全看到它如何解决组织原本用计划来解决的其他问题。选择的需求仍然存在。在某些方面，随着构建成本下降，这些问题反而会变得更重要。如果做更多东西变得更容易，那么做错东西也会变得更容易。

这种转变里仍然有一些真实的东西。能力变化很快，市场偏好也在流动。非常长的计划周期现在风险更高，因为在你完成计划之前，市场可能已经变了。

同时，也确实有更强的理由去做更多实验。AI 能力持续提升，而我们对如何使用它们的理解还不完整。有价值的用例、工具和工作流往往必须从尝试中浮现出来。它们并不总是能被提前规划。

Linear 从一开始就是这样运作的。我们至少会有六个月的计划，用来设定整体方向。但我们保留在任何一个月、任何一周改变计划或优先级的能力。计划始终只是一个关于“什么似乎值得做”的信念。随着我们学习到新东西，或注意到变化，我们就调整它。现在我们也在鼓励更多实验，给新出现的用例留出更多空间，而不是只依赖计划。

但是，没有计划、没有规划、没有方向，或者只有非常短的路线图，也会让组织失去舵。风险会变成：为错误的客户、出于错误的原因，构建错误的东西。

如果没有思考、方向，或者没有一个关于你真正想达成什么的计划，你最终可能会做那些最容易做的事。当你不做计划这项练习时，你也永远不会让自己的思考、承诺、计划或时间线失败，因为你从未真正制定它们。你可以随便构建一些东西，看看接下来会发生什么，并且可能让 AI 把你带向最容易的方向，而不是最重要的方向。

## 你引导 AI，AI 也引导你

这触及了我以前写过的东西：工具总会引导并影响工作流。它们总会向你展示一条路径。

AI 工具不同，因为它们是思维工具，而不只是机械工具。它们引导你和你的工作的能力，比以前的工具更强。

有一种有效的工作流，是你不与 AI 对抗，而是让它牵引你。这本质上可以是做得好的 vibe coding：顺着工具的纹理走，快速推进，接受模型有自己的方向，并利用这种动量。

这可能很有用，哪怕它并不完全把你带到你原本想去的地方。至于这是不是好事，取决于具体情况。

## 专业能力悖论

AI 往往在你最不了解的领域里显得最令人印象深刻。我认为这是市场叙事产生认知失调的最大原因。对普通观察者来说，AI 能力常常被描述和理解成没有边界。

在你深度理解的领域里，你会看到缺口。输出很接近，但并不完全正确。它遗漏上下文，编造细节，选择显而易见的路径，或者需要大量引导才变得有用。

在你了解较少的领域里，同样的输出可能像魔法，因为你缺乏判断力，看不出缺了什么。这是一种大规模的 Gell-Mann Amnesia 和邓宁-克鲁格效应。

悖论在于：专业能力让 AI 更难使用，但如果你知道如何正确驾驭它，也会让 AI 更有价值。专家看得见那些“AI 味”的粗糙内容，但他们也知道如何引导、约束和评估模型。

所以，AI 并没有移除专业能力的价值。它让专业能力更多地变成方向感、判断力，以及知道什么才算好的能力。

## 编程

和客户以及业内公司交流时，可以明显感觉到，agentic coding 现在已经相当标准化了。但这里有一个光谱。私下里几乎没有人说他们的 agent 写了 100% 的代码。我也没有听到很多真实公司在运行大规模独立 agent 群的真实故事。

工程师仍然非常深度地参与 agent 的引导，而且通常可能一次也只能运行几个 agent，同时在后台跑少量云端 agent 来处理更琐碎的修复。再多的话，你就进入了《哈里森·伯吉朗》的世界：总有东西不停 ping 你，打断你的思考过程。

我们在 Linear 也看到了这一点。现在我们大多数付费 workspace 都安装了 coding agent，使用这些 agent 的活动量在几个月内增长了超过 5 倍。我并不知道每家公司内部到底如何使用云端 agent，但方向很清楚：这些工具已经成为正常开发工作流的一部分。

在 Linear，工程师会使用各种 agent，我们也确实看到速度提升。仅用我们自己的云端 coding agent，我们现在每个月修复超过 1,000 个 issue，而且增长很快。我看到的最大影响是带宽增加。我们可以处理比以前更多的小问题。那些可能太小、太烦、太耗时的工作，现在变得更容易完成。

但困难的问题仍然困难。Agentic coding 并不会像叙事经常暗示的那样大幅加速这些问题。它在边缘部分有帮助：搭脚手架、重构、调查、做小修复。但困难的部分仍然需要理解系统、做权衡，以及知道应该存在什么。

在专家级编程工作流里，价值通常不是“AI 写代码，你接受”。价值在于围绕工作本身的加速。它可以探索方案、搭建实现、调试错误、重构代码、编写测试或大型迁移，并处理较小的修复。

专家仍然提供品味、约束和最终判断。

## 设计

新的图像生成和设计能力已经好很多，但我仍然觉得它们很难用。

图像生成似乎会随着迭代次数增加而退化。很难让 AI 以某一种特定方式只改变某一个特定东西。它经常一次改变很多东西。每一次迭代似乎也会给整张图再叠上一层某种滤镜；为了修好它，你可能不得不重新开一个聊天，重新写描述，重新上传文件。

我希望看到更好的工具，可以把 AI 限制在特定区域内，而不是每次都允许它重新解释整个输出。写作里也会发生这种情况：要求做一个修改，模型却常常重塑整篇文章。我希望图像生成有更好的控制方式和工作流。

## 设计与代码

我一直在思考，让设计工具直接作用于生产代码库这件事到底有什么价值。

许多设计 + 代码工具会展示一个 React 组件，比如一个按钮，然后让你用设计工具修改它。但当我在设计时，我可以想象这个按钮如何工作。在那个阶段，它是一个矢量形状还是一个 React 按钮，对我来说并没有那么重要。

我做的很多设计工作并不是生产设计。我不是在试图实现最终版本，也不是要测试每一个边界情况。大多数设计工作是为了做决策、理解问题、找到契合点。这个过程会产生很多变体和凌乱的想法。

我不希望每一次修改都带着生产代码的速度成本或 token 成本。我不希望因为提示没有完成而陷入错误。我也不想主要通过文字或代码来设计。设计是一个视觉领域，对我来说，它在视觉化、基于画布的工具里仍然最有效。

即使我们暂时放下“是否还需要 UI”这个争论，设计仍然是一种塑造问题和解决方案的方式。大多数设计工作，是理解一个功能如何嵌入现有系统，如何与其他概念互动，它的边界在哪里，以及它要为什么而优化。

即便一个产品是通过 MCP、CLI 或 API 被使用，它的概念和工作流仍然需要以某种方式说得通。

我会觉得更有用的，是高级视觉工具，或者语义化 UI 设计工具。你可以更直接地定义布局和模式。你画的不是一个矩形，而是一个 modal，它可以继承系统中的某些方面。AI 可以帮助填充屏幕，生成变体，探索方向。价值在于这些模式，而不在于这些模式是否背后已经连接了真实生产代码。

然后，当我真的有了想要在线原型化的东西时，应该有一个清晰的工作流，把设计文件提炼给 AI，并把它翻译成代码。在 AI 世界里，设计对我来说像规划。它就像 `/gstack` 在运行，向你提问；只不过那是你自己的大脑在提问，并在屏幕上寻找解决方案。

## 领域很重要

对于任何关于一家公司如何使用 AI 的说法，我都会试着问：他们是谁？他们所在的领域是什么？他们处于什么阶段？他们的产品是什么？他们有客户吗？客户又是谁？

我一直认为，不同产品需要不同层级的稳定性、信任、安全性，甚至不同层级的产品设计。每个产品都需要不同程度、不同类型的设计。

像 email 这样高频、触感强的工具，需要大量 UX 打磨，因为用户会感受到每一个小割伤。他们一直在使用它。小摩擦会复利。

后端服务则不同。价值可能在后端逻辑里。UI 可以更有限，甚至更粗糙，但产品仍然可能有价值。

我认为许多 AI 公司更像后端公司。能力在模型里。外层的 harness 和工具在模型之上迭代，但到目前为止，它们常常处在幕后。这帮助它们快速移动，因为每个功能都可以成为系统中的另一个工具，而不需要太多视觉或概念 footprint。

这感觉更接近经典 UNIX 系统：程序是基于文本的，可以被独立创建，并在运行时组合。与现代 macOS 之类的系统相比，这种操作系统的可组合性更容易实现。

---

最后一个论点总是：“这些工具现在就是它们未来最差的样子。”

我同意，但我试着贴近它们现在的能力，因为我并不知道它们究竟会变成什么。这就是我谨慎乐观的来源。我不觉得做末日论者有什么价值，但我也不觉得一厢情愿有什么价值。

更有用的位置，是仔细观察、尝试事物，并保持判断力。AI 显然正在改变很多东西，但我不认为我们今天可以只靠更用力地相信来构建。我们需要更真实，注意到现在什么是可能的，决定什么值得做，并随着现实变化而调整。

就像那首老童谣大概会说的：如果潜力就是收入，那么所有这些资本开支早就已经是利润了。
